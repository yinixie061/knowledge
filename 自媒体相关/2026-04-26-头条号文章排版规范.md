# 头条号文章排版

## 概述

今日头条/头条号的富文本编辑器比微信更严格，会过滤掉大量 HTML 标签和 CSS 样式。本 skill 生成的是"语义极简 HTML"——只保留头条一定能识别的标签，放弃所有视觉装饰，靠标签本身的语义在编辑器中呈现格式。

## 核心约束

- **不支持的标签**（会被过滤或转纯文本）：`<div>`、`<section>`、`<table>`、`<tr>`、`<td>`、`<style>`、`<script>`、`<iframe>`、`<form>`、`<input>`
- **不支持的样式**：所有 `style="..."` 内联样式大概率被 strip，**不要依赖 CSS**
- **支持的标签**（保留语义）：`<h1>`-`<h6>`、`<p>`、`<b>` / `<strong>`、`<i>` / `<em>`、`<ul>` / `<ol>` / `<li>`、`<blockquote>`、`<img>`、`<a>`、`<br>`、`<hr>`
- **图片**：本地图片无法直接粘贴，生成后需手动上传到头条号
- **表格**：头条号编辑器不支持表格，**所有表格必须转文字列表**

## 排版策略

不靠 CSS，只靠标签语义：

| 效果 | 实现方式 |
|------|----------|
| 文章标题 | `<h1>` |
| 章节标题 | `<h2>`、`<h3>` |
| 正文段落 | `<p>` |
| 加粗强调 | `<b>` 或 `<strong>` |
| 引用/金句 | `<blockquote>` |
| 有序列表 | `<ol><li>` |
| 无序列表 | `<ul><li>` |
| 分隔线 | `<hr>` |
| 换行 | `<br>` |
| 图片 | `<img src="URL">` |
| 链接 | `<a href="URL">` |

## 表格处理规则

所有表格必须转换为文字列表形式：

```html
<!-- 原表格 -->
<table>...</table>

<!-- 转换为 -->
<p><b>表格标题</b></p>
<ul>
  <li><b>表头1</b>：值1 | <b>表头2</b>：值2</li>
  <li><b>表头1</b>：值3 | <b>表头2</b>：值4</li>
</ul>
```

或用段落形式：

```html
<p><b>第一行：</b>字段A=xxx，字段B=xxx</p>
<p><b>第二行：</b>字段A=xxx，字段B=xxx</p>
```

## 引用卡片处理

原公众号的引用卡片（带背景色、边框）在头条号中只能用 `<blockquote>` 代替：

```html
<!-- 公众号版 -->
<section style="background:#fef5f5; border-left:4px solid #c0392b; padding:16px;">
  <p>金句内容</p>
</section>

<!-- 头条版 -->
<blockquote>
  <p><b>金句内容</b></p>
</blockquote>
```

## 完整工作流

### Step 1：读取源文件
- 读取用户提供的 Markdown 文件（`.md`）或文本
- 理解文章结构和内容

### Step 2：内容转换
- 将 Markdown 转换为极简 HTML
- **所有表格转列表/段落**
- **去掉所有 style 属性**
- **用 `<blockquote>` 代替引用卡片**
- 保留章节层级（h1-h3）
- 保留加粗、列表、链接

### Step 3：生成工具页
- 创建带"复制到头条"按钮的 HTML 页面
- 使用 ClipboardItem API 同时写入 `text/html` 和 `text/plain`
- 页面结构：工具栏 + 文章内容区（id="articleContent"）+ 复制脚本

### Step 4：交付
- 输出 `.html` 工具页到工作区
- 提醒用户图片需手动上传到头条号编辑器
- 建议发布流程：点复制按钮 → 打开头条号编辑器 → Ctrl+V 粘贴 → 手动上传图片

## HTML 模板结构

```html
<!DOCTYPE html>
<html><head><meta charset="UTF-8"></head><body>
<div class="toolbar">
  <button onclick="copyRichText()">复制到头条编辑器</button>
</div>
<div id="articleContent">
  <h1>文章标题</h1>
  <p><b>作者/引言</b></p>
  <hr>

  <h2>一、章节标题</h2>
  <p>正文段落...</p>
  <blockquote><p><b>金句引用</b></p></blockquote>
  <ul>
    <li>列表项一</li>
    <li>列表项二</li>
  </ul>

  <h2>二、章节标题</h2>
  <p>（原表格内容转列表）</p>
  <p><b>表头说明</b></p>
  <ul>
    <li><b>字段A</b>：值1，<b>字段B</b>：值2</li>
  </ul>

  <hr>
  <p><i>免责声明/结语</i></p>
</div>
<script>
async function copyRichText() {
  const content = document.getElementById('articleContent');
  const htmlContent = content.innerHTML;
  const textContent = content.innerText;
  if (navigator.clipboard && window.ClipboardItem) {
    const htmlBlob = new Blob([htmlContent], { type: 'text/html' });
    const textBlob = new Blob([textContent], { type: 'text/plain' });
    const item = new ClipboardItem({ 'text/html': htmlBlob, 'text/plain': textBlob });
    try {
      await navigator.clipboard.write([item]);
      alert('已复制！去头条号编辑器按 Ctrl+V 粘贴');
      return;
    } catch (err) {
      console.log('Clipboard API failed, falling back', err);
    }
  }
  const range = document.createRange();
  range.selectNodeContents(content);
  const selection = window.getSelection();
  selection.removeAllRanges();
  selection.addRange(range);
  document.execCommand('copy');
  selection.removeAllRanges();
  alert('已复制！去头条号编辑器按 Ctrl+V 粘贴');
}
</script>
</body></html>
```

## 与公众号版的区别

| 特性 | 公众号版 | 头条号版 |
|------|----------|----------|
| 标签 | `<section>`、`<table>`、`<div>` | 只用 `<h1>`-`<h6>`、`<p>`、`<ul>`/`<ol>`、`<blockquote>` |
| 样式 | 内联 `style="..."` | **无 style**，纯标签语义 |
| 表格 | `<table>` 正常显示 | **转文字列表** |
| 引用卡片 | 带背景色边框的 `<section>` | `<blockquote>` |
| 配色 | 红/蓝/绿等主题色 | 不设置，靠编辑器默认 |
| 多栏布局 | `<table>` 双栏 | 不使用，单栏流式 |

## 常见陷阱

1. **写了 style 属性**：头条会 strip 掉，写了也白写
2. **用了 `<table>`**：表格可能直接消失或变乱码
3. **用了 `<div>` 或 `<section>`：可能被过滤或转 `<p>`
4. **本地图片**：剪贴板复制无法携带图片，需手动上传
5. **复杂的嵌套结构**：尽量扁平，减少嵌套层级

- 浏览器 —— 预览和复制
- 头条号编辑器 —— 最终发布
