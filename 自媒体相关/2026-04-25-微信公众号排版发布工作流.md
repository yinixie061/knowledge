# 微信公众号文章排版与发布工作流

## 概述

微信公众号文章使用受限的富文本编辑器，有特定的 HTML/CSS 限制。本文档覆盖完整工作流：内容排版、图片生成、产出可发布的最终输出。

**核心约束：**
- 微信编辑器仅支持 **内联样式**（`style=""` 属性），`<style>` 标签会被过滤
- **不支持 `display: flex` / `display: grid` / `position` / `float`** —— 多栏布局用 `<table>`
- 图片必须上传至微信 CDN（`mmbiz.qpic.cn`），本地/外部 URL 无法渲染
- **暗黑模式**：微信自动反色，避免深色背景（`#1a1a2e`、`#000` 等），使用浅色背景
- 剪贴板复制无法携带本地图片（浏览器安全限制）
- 支持的标签：`<section>`、`<p>`、`<span>`、`<strong>`、`<em>`、`<a>`、`<img>`、`<table>`、`<ul>`、`<ol>`、`<li>`、`<h1>`-`<h6>`、`<hr>`
- 不支持：`<script>`、`<iframe>`、`<form>`、`<input>`、`<style>`

## 速查表

| 任务 | 方法 |
|------|------|
| 读取文章内容 | 读取上传的文件（`.md`、`.txt`、`.docx`） |
| 生成图片 | 使用 `GenerateImage` 工具生成封面+插图 |

| 预览 HTML | 创建带复制按钮的工具页 |

## 图片生成

### 封面图
- **尺寸**：`landscape_16_9`（1280x720）—— 微信推荐比例
- **风格**：贴合文章主题，专业简洁
- **Prompt 模板**：`"WeChat article cover image: [主题描述]. Style: modern minimalist illustration, clean background, [配色方案]. Purpose: article cover thumbnail for social media."`

### 正文插图
- **尺寸**：`landscape_16_9`（1280x720）
- **数量**：2-4 张，放置在关键章节分界处
- **风格**：与封面保持一致，偏轻/装饰性
- **位置**：主要章节后（通常在第 1、4 节后和结语前）

### 图片规则
- 保存到工作区：`workspace/images/`
- 使用描述性文件名：`cover.jpg`、`illust1.jpg`、`illust2.jpg` 等
- **不要** 在图片下方添加说明文字（用户可能要求删除）
- 图片需手动上传至微信 —— 提供清晰说明

## 微信兼容 HTML 排版

### 关键规则

**1. 所有样式必须内联**
```html
<!-- 正确 -->
<p style="font-size:15px; color:#333; line-height:1.9;">Text</p>

<!-- 错误 —— style 标签会被过滤 -->
<style>.text { font-size: 15px; }</style>
<p class="text">Text</p>
```

**2. 使用 `<section>` 代替 `<div>`**
```html
<!-- 正确 -->
<section style="padding:20px;">Content</section>

<!-- 错误 —— div 可能被过滤 -->
<div style="padding:20px;">Content</div>
```

**3. 禁用 flex/grid —— 用 `<table>` 做多栏**
```html
<!-- 正确 —— table 双栏布局 -->
<table style="width:100%; border-collapse:collapse; border:none;">
  <tr>
    <td style="width:48%; background:#f0f4ff; padding:16px; border:none;">Left</td>
    <td style="width:4%; border:none;"></td>
    <td style="width:48%; background:#fff0f0; padding:16px; border:none;">Right</td>
  </tr>
</table>
```

**4. 暗黑模式兼容 —— 避免深色背景**
```html
<!-- 正确 —— 浅色背景+彩色边框 -->
<section style="background:#fef5f5; border:2px solid #c0392b; border-radius:8px; padding:20px;">
  <p style="color:#c0392b; font-weight:bold;">Key quote</p>
</section>

<!-- 错误 —— 深色背景在暗黑模式下崩坏 -->
<section style="background:#1a1a2e; padding:20px;">
  <p style="color:#fff;">Key quote</p>
</section>
```

**5. 图片必须用响应式尺寸**
```html
<img src="IMAGE_URL" style="max-width:100%; height:auto; display:block;" alt="description">
```

### 推荐样式系统

| 元素 | 样式 |
|------|------|
| 正文 | `font-size:15px; color:#333; line-height:1.9; text-indent:2em;` |
| 章节标题 | `font-size:19px; font-weight:bold; color:#1a1a1a;` |
| 文章标题 | `font-size:24px; font-weight:bold; color:#1a1a1a; line-height:1.5;` |
| 强调加粗 | `<span style="color:#c0392b; font-weight:bold;">text</span>` |
| 引用卡片 | `background:#fef5f5; border-left:4px solid #c0392b; padding:16px 20px; border-radius:0 8px 8px 0;` |
| 金句卡片 | `background:#fef5f5; border:2px solid #c0392b; border-radius:8px; padding:20px;` |
| 列表卡片 | `background:#fafafa; border-radius:8px; padding:20px;` |
| 列表项 | `padding-left:16px; text-indent:-16px;` + 项目符号前缀 |
| 分隔线 | `<hr style="border:none; border-top:1px solid #eee; margin:10px 0;">` |
| 章节编号 | 内联圆：`width:28px; height:28px; background:#c0392b; color:#fff; border-radius:50%; text-align:center; line-height:28px;` |
| 表头 | `background:#c0392b; color:#fff; font-weight:bold; padding:10px 8px;` |
| 表格单元格 | `padding:10px 8px; color:#444; line-height:1.6; border-bottom:1px solid #f0f0f0;` |
| 免责声明 | `font-size:13px; color:#999; font-style:italic; text-align:center;` |

## 发布工作流

### 推荐：doocs/md 编辑器（图片最佳方案）

行业标准工作流：

1. **准备 Markdown**：使用原始文章内容的 Markdown 格式
2. **打开编辑器**：访问 [https://doocs.github.io/md/](https://doocs.github.io/md/)
3. **粘贴内容**：将 Markdown 文本粘贴进编辑器
4. **上传图片**：使用编辑器的图片上传功能（直接上传至微信 CDN）
5. **选择主题**：从主题库选择视觉主题
6. **复制**：点击复制按钮，然后粘贴到微信编辑器（Ctrl+V）
7. **发布**：预览并发布

### 备选：直接粘贴 HTML

纯文字文章或图片已在微信 CDN 上时适用：

1. 创建全内联样式的微信兼容 HTML
2. 在浏览器中打开 HTML 文件
3. 全选（Ctrl+A）复制（Ctrl+C）
4. 粘贴到微信编辑器（Ctrl+V）

**限制**：本地图片无法复制，仅适用于微信 CDN URL。

### 备选：一键复制工具页

使用 Clipboard API 创建带复制按钮的 HTML 页面：

```html
<!DOCTYPE html>
<html><head><meta charset="UTF-8"></head><body>
<div class="toolbar">
  <button onclick="copyRichText()">Copy to WeChat Editor</button>
</div>
<div id="articleContent">
  <!-- WeChat-compatible HTML content here -->
</div>
<script>
function copyRichText() {
  const content = document.getElementById('articleContent');
  const range = document.createRange();
  range.selectNodeContents(content);
  const selection = window.getSelection();
  selection.removeAllRanges();
  selection.addRange(range);
  document.execCommand('copy');
  selection.removeAllRanges();
}
</script>
</body></html>
```

**限制**：无法复制本地图片。

## 完整工作流（逐步）

### Step 1：理解需求
- 读取文章内容（来自上传文件或粘贴文本）
- 确认：输出格式（HTML / Markdown / 两者）、图片需求、配色主题
- 检测文章语言并全程保持

### Step 2：生成图片（如需）
- 使用 `GenerateImage` 工具生成 1 张封面 + 2-4 张插图
- 保存到 `workspace/images/`
- 全篇图片风格统一

### Step 3：创建格式化输出
- **主输出**：生成 Markdown 版本（用于 doocs/md 工作流）
- **次输出**：生成微信兼容 HTML（用于参考/预览）
- 应用上方样式系统的一致性样式
- 确保暗黑模式兼容（无深色背景）

### Step 4：交付并附说明
- 所有输出文件放入工作区
- 包含发布工作流的清晰说明
- 提醒用户图片需手动上传至微信

## 常见陷阱

1. **暗黑模式下的深色背景**：微信会反色 CSS 颜色，始终使用浅色背景
2. **Flex/Grid 布局被过滤**：用 `<table>` 代替
3. **外部图片不加载**：微信屏蔽非 `mmbiz.qpic.cn` 的 URL
4. **剪贴板无法携带图片**：图片必须单独上传
5. **`<style>` 标签被过滤**：所有 CSS 必须内联
6. **`<div>` 可能被过滤**：优先使用 `<section>`
7. **高饱和度颜色**：使用柔和色调以兼容暗黑模式
8. **阴影被反色**：完全避免 box-shadow
9. **白边图片**：使用无边框图片或透明 PNG
10. **务必测试暗黑模式**：在亮色和暗色模式下都预览
